# Execution Plan: Fix Message Persistence in Engine and Tests

## Objective
Ensure that messages generated by the assistant via `Engine.ask()` and within skills are correctly appended to the `Context` so they persist in the conversation history, fixing the infinite loop with `test_lingo_when_filter` and `test_lingo_session_resumption`.

## Architectural Impact
The `Engine` class provides low-level LLM operations. While `Engine.reply()` remains a pure functional call (returning a message without side effects on the context), higher-level composite methods like `Engine.ask()` should manage context state to ensure the "question" part of the interaction is recorded. Skills using `Engine.reply()` directly are responsible for appending the result to the context, or they should use `Flow` nodes that handle this automatically.

## File Operations

### 1. `lingo/engine.py`
- **Modify** `Engine.ask(self, context: Context, question: str)`:
  - Capture the result of `await self.reply(context, question)`.
  - Append the resulting `Message` to `context`.
  - Then call `await self.input()`.

### 2. `tests/test_core.py`
- **Modify** `test_lingo_when_filter`:
  - Update the `stop_filter` skill to append the response: `ctx.append(await eng.reply(ctx, "Stopped."))`.
- **Modify** `test_lingo_session_resumption`:
  - While the fix in `Engine.ask` covers the first part, ensure the skill logic correctly appends the second reply: `ctx.append(await eng.reply(ctx, f"Age is {age}"))`.

## Step-by-Step Execution

### Step 1: Update Engine Logic
Update `lingo/engine.py` to make `ask` stateful:
```python
async def ask(self, context: Context, question: str) -> str:
    response = await self.reply(context, question)
    context.append(response) # Now the question is in the context
    return await self.input()
```

### Step 2: Fix the Filter Test
Update `tests/test_core.py` to reflect that `eng.reply` in a custom skill (FunctionalNode) requires manual appending:
```python
@bot.when("user says stop")
async def stop_filter(ctx, eng):
    nonlocal filter_executed
    filter_executed = True
    msg = await eng.reply(ctx, "Stopped.")
    ctx.append(msg) # Explicitly append
```

### Step 3: Fix the Session Resumption Test
Update `tests/test_core.py` for `test_lingo_session_resumption`:
```python
@bot.skill
async def age_skill(ctx, eng):
    age = await eng.ask(ctx, "What is your age?")
    msg = await eng.reply(ctx, f"Age is {age}")
    ctx.append(msg) # Explicitly append
```

### Step 4: Verify Lingo.chat
Review `Lingo.chat` in `lingo/core.py` to ensure it correctly returns the last assistant message. The current implementation already iterates backwards to find the first message with `role == "assistant"`, which is correct.
